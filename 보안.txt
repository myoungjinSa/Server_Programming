*패킷 암호화 :
1. 피씨방에서 기지국까지의 패킷은 정부나 kt에서 책임지는게 아님.
2. 중국에서 들어오는 접속은 막지만, 국내 ip 접속은 막지 못함
3. 비정상 접속이라 판단되면 접속을 끊어야함.


*nProtect와 같은 해킹 감지 프로그램을 게임회사에서 설치해서 클라이언트와  
처음 패킷을 주고받는거는 nProtect가 클라이언트에 설치되었는지 확인하는 패킷을 주고받는다.

*서버 부정 접속 방지 :
- 서버 컴퓨터 관리 철저 (fire wall 설치해야함)
- 디버깅용 백도어 삭제 - (개발자 용도의 teleport 같은 치트키)
- 해킹의 기본이 stack overflow(모든 텍스트 길이 관리)
- 특수 문자 대체 -> DB 쿼리 인젝션 -> ";Update gold : 10000 in user, table"  "를 %20 아스키코드로 바꿔서 넣어야함



-개발자 컴퓨터 해킹 
-> 방지 대책 : 망 분리, USB 사용 금지 -> 읽는건 되는데 쓰는게 금지

->요즘은 개발자용 PC에는 데이타가 없다. 클라우드 서버에 데이터가 있고 개발자용 PC는
  클라우드 서버에 접속해서 작업을 한다.


*AUTO 프로그램 해결방법

-클라이언트 검증
1. 특수 패킷 -> 	AUTO인지 아닌지 를 물어보는 패킷을 몰래 섞어서 보낸다. 
   		-> AUTO프로그램이라고 판단이 됐다고 바로 정지하면 안됨 -> 그러면 해당 패킷이 존재하는게 쉽게 발각됨
		->그렇기 때문에 랜덤한 시간에 정지를 시킨다. 

2. 클라이언트 내부 메모리 검사 



-클라이언트 검사는 계속 업데이트 되어야함 -> 그렇기 때문에 AUTO 프로그램이 비싼가격을 유지하고
